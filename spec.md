# SmartProcure AI 驅動企業採購智慧決策系統 - 軟體規格書 (PRD / Spec)

## 1. 產品概述 (Product Overview)
**SmartProcure** 是一個專為企業採購戰略部打造的決策輔助平台。藉由引入機器學習預測模型 (節省率預測、風險評估) 與可解釋性 AI (SHAP)，系統旨在將採購防呆從「事後審核」提升為「事前預警與動態推薦」，藉此極大化企業的採購節省金額 (Savings) 並降低供應鏈風險。

## 2. 系統架構 (System Architecture)
* **前端架構**：採用 HTML/CSS/JS 開發之**單頁面應用程式 (SPA)**。
  * **UI 設計風格**：深色模式 (Dark Mode) 結合毛玻璃特效 (Glassmorphism)，強調現代感與戰略層級的視覺體驗。
  * **動態切換**：透過 `data-target` 屬性實現五大分頁的無縫隱藏/顯示切換。
* **資料串接設計**：前端所有具備動態資料屬性的 DOM 節點均預留專屬 `id` (例如 `id="val-est-savings"`)，以便後端 API 或 Python 腳本直接更新 (Inject) 資料。

## 3. 介面與功能規格 (UI & Functional Specifications)

本系統涵蓋五大核心分頁，可透過左側導覽列切換：

### 3.1 總覽頁 (Overview)
* **目標**：提供採購主管全域的績效與風險監控。
* **關鍵指標 (KPI)**：
  * **預估節省金額**：呈現本季/本月預估能省下的總金額及趨勢變化 (上揚/下跌)。
  * **平均風險分數**：彙整所有進行中訂單的整體風險指數，並標示狀態燈號 (安全/警告)。
  * **高風險訂單數**：統計被模型判定為「超支預算」或「交期延宕」的高風險訂單數量。
* **視覺化圖表**：預留趨勢圖表區塊，後續可串接各類別採購的預算消耗比例。

### 3.2 議價預測頁 (Bargain Prediction)
* **目標**：讓採購人員在下單或議價前，輸入條件以預測潛在的節省率 (Savings Pct)。
* **輸入欄位 (Input Form)**：
  * **供應商 (Supplier)**：下拉選單選擇歷史供應商。
  * **合約類別 (Contract)**：下拉選單選擇 (如：年度框架協議、單次採購)。
  * **數量 (Quantity)**：數值輸入。
  * **預算金額 (Budget - USD)**：數值輸入。
* **輸出結果 (Prediction Result)**：
  * 點擊「執行預測」後，經由加載動畫展示預測結果。
  * 顯示預測 **節省率 (+/- %)** 與 **預估金額**。
  * 顯示模型判斷理由標籤 (如：數量達折扣門檻、歷史議價空間大)。

### 3.3 IT Software 供應商推薦模型 (Supplier Recommendation Model)
* **目標**：針對同一採購類別，根據多維度資料推薦較適合優先考慮的供應商。
* **推薦指標**：
  * **歷史節省率 (Savings Pct)**
  * **準時交付率 (On-Time Delivery)**
  * **供應商風險 (Supplier Risk)**
  * **ESG 分數 (ESG Score)**
  * **優先供應商狀態 (Preferred Supplier)**
* **功能清單**：
  * **情境切換按鈕**：提供「降本導向 (Cost-driven)」、「急件採購 (Urgent)」、「合規導向 (Compliance-driven)」按鈕。
  * **動態排序**：點擊情境後，動態調整推薦權重，即時更新推薦名次與推薦亮點。

### 3.4 供應商分析頁 (Supplier Analysis)
* **目標**：多維度評估與管理供應商，以打破單一來源依賴並優化合規性。
* **功能清單**：
  * 以資料表 (Data Table) 列出供應商清單。
  * **欄位包含**：供應商名稱、Tier (等級)、Region (地區)、ESG Score (企業永續評分)、歷史爭議紀錄 (Controversies)。
  * 依照不同表現，將 Tier、ESG 與爭議分數套用對應的紅綠燈號與發光特效。

### 3.5 報表頁 (Reports)
* **目標**：自動化生成採購績效報告，減少手動統整時間，並提供模型決策的透明度。
* **功能清單**：
  * 支援「產出採購週報」與「產出採購月報」匯出按鈕。
  * **報表預覽區塊**：展示文字摘要 (Text Summarization)，包含攔截筆數、省下總額。
  * **AI (SHAP) 決策摘要**：將技術性的 SHAP 值轉化為自然語言，解釋本週重大擋單的背後原因。

### 3.6 系統日誌與稽核 (System Audit Logs)
* **目標**：確保系統內的所有關鍵操作皆具備可追溯性 (Traceability)，符合企業內部稽核與資安規範。
* **功能清單**：
  * **操作追蹤**：任何針對訂單、供應商狀態的「更新 (Update)」與「刪除 (Delete)」操作，都必須強制記錄至系統後台 Log 日誌。
  * **日誌可視化**：系統需提供專屬的「稽核日誌 (Audit Log) 檢視面板」(可獨立頁面或整合於報表/設定頁)，讓管理員可以隨時查看：
    * 操作者帳號 (User ID / Role)
    * 操作時間 (Timestamp)
    * 變更前與變更後的資料對比 (Before / After Snapshot)
    * 操作原因 (如適用)

## 4. 機器學習模型戰略整合 (Model Strategy Integration)

本平台依賴兩大核心模型，其參數調校與觸發規則遵循 `model_tuning_playbook.md`：

### 模型 1：採購節省率預測模型
* **對應分頁**：議價預測頁、風險訂單頁。
* **演算法目標**：預測每筆採購的潛在節省比例，並區分為「節省 (>5%)」、「接近預算 (-2% 至 5%)」、「超出預算 (<-2%)」。
* **特徵考量**：必須排除實際支付單價以防止目標洩漏 (Target Leakage)，完全依賴 PO 開立前可取得之資訊。
* **監控退場**：若 RMSE 誤差上升超過 10%，介面將於頂部觸發 Alert Banner 提示管理員調降 Learning Rate 並重訓。

### 模型 2：IT Software 供應商推薦模型
* **對應分頁**：供應商分析頁。
* **演算法目標**：依據歷史績效推薦供應商，可根據業務需求動態調整權重。
* **情境切換** (未來擴展功能)：
  1. **降本導向**：提高歷史節省率權重。
  2. **急件採購**：提高準時交付率與 Lead Time 權重。
  3. **合規導向**：極大化 Preferred Supplier 與 ESG Score 權重。

## 5. 未來擴展與 API 串接指南 (API Hooks)

前端已為以下關鍵資料節點預留 `id`，後端開發者可直接透過 DOM 替換或框架 (Vue/React) Data Binding 更新：

* `val-est-savings`: 預估節省金額 (總覽頁)
* `val-avg-risk`: 平均風險分數 (總覽頁)
* `val-high-risk-count`: 高風險訂單數 (總覽頁)
* `val-pred-savings` & `val-pred-amount`: 預測結果數值 (議價預測頁)
* `risk-table` & `supplier-table`: 表格資料本體 (風險訂單/供應商頁)
* `val-report-content`: LLM 生成的動態報告摘要內容 (報表頁)
