## 頁面 1：開場與設計初衷 (The Why)

**標題：當強大的 AI 遇上傳統採購系統...**

- **痛點分析**：
  - 開發初期我們意識到一個重大問題：模型組辛苦訓練出準確率極高的 XGBoost 模型，但如果丟給終端使用者（採購人員）的只是一堆 JSON 數據或冰冷的 CSV 報表，這套系統注定失敗。
  - 傳統 ERP 系統的操作邏輯繁瑣，且與現代 AI 的即時互動性嚴重脫節。
- **我們的任務 (UI 組使命)**：
  - 我們要在「生硬的 AI 模型」與「人類決策者」之間，搭建一座最強的橋樑。
  - 將這套系統定位為採購人員的**「全知決策輔助儀表板 (Decision Support Dashboard)」**，而不只是一個查資料的網頁。

## 頁面 2：從 0 到 1 的演進歷程 (The Journey)

**標題：無中生有：我們的迭代與進化之路**

我們並不是一開始就畫出現在的畫面，而是經歷了不斷的打磨：
1. **概念成型 (Wireframing)**：
   - 最一開始，我們只有黑白線框圖，確立了「左側導覽、上方搜尋、右側動態顯示」的經典 SPA 佈局。
2. **導入現代美學 (Cyberpunk & Glassmorphism)**：
   - 為了凸顯這是一套「次世代 AI 系統」，我們大膽拋棄了傳統死板的企業藍白配色，導入了深色模式 (Dark Mode)、科技螢光漸層，以及極具質感的毛玻璃透視效果 (Glassmorphism)，讓長官一看就有 WOW 的感覺。
3. **細節淬鍊 (Micro-interactions)**：
   - 從左下角的「艾鍗學院動態發光 Logo」，到選單切換時的霓虹殘影，我們在無數個細節中加入了微動畫，讓系統看起來像是有生命一樣在運作。

## 頁面 3：融入 Domain Knowledge 的 UX 設計

**標題：不只好看，更要「好用」— Vendor 360° 計分卡**

設計不能只有空殼，我們將「採購領域知識 (Domain Knowledge)」深度融入了 UX 設計中：
- **痛點**：長官問「這家廠商表現如何？」時，過去要跨好幾個頁面查資料。
- **解法 - Global Search + Vendor 360**：
  - 我們復活了全域搜尋列，打造成「一鍵透視鏡」。
  - 搜尋後瞬間跳出的不是無聊的表格，而是四大核心 KPI 卡片（歷史總花費、平均節省率、準交能力、ESG 評分）。
- **創新 - RPG 角色能力值雷達圖**：
  - 業界的雷達圖常常讓人看不懂。我們首創將「成本、交期、ESG」轉化為直覺的三角頂點。讓採購人員就像在看遊戲角色能力值一樣，一眼秒懂 A 廠商和 B 廠商誰是「成本導向」，誰又是「品質導向」！

## 頁面 4：打造流暢的企業級工作流 (Seamless UX)

**標題：拒絕妥協的完美體驗 — 滑出式建檔面板 (Slide-out Drawer)**

這是一個我們針對「細節與穩定度」絕不妥協的實際案例：
- **挑戰**：一開始設計「新增供應商」功能時，我們採用了傳統的置中彈出視窗 (Modal) 並加上超炫的毛玻璃特效。但我們在深度測試後發現，當它覆蓋在龐大且密集的圖表數據 (Canvas) 之上時，極易引發硬體加速 (GPU) 崩潰，導致畫面死當。
- **進化與轉念**：我們沒有選擇閹割畫面的圖表，也沒有放生這個 Bug，而是果斷升級為現代企業後台 (如 Azure, AWS) 最主流的 **右側滑出式建檔面板 (Drawer)**。
- **成效**：這不僅 `100%` 解決了畫面卡死的問題，使用者現在還能一邊看著左側的首頁核心數據，一邊在右側面板流暢地建立新廠商資料，完美達成了「無縫工作流」！

## 頁面 5：AI 雷達圖精準化與分數一致性 (Scoring Integrity)

**標題：分數不能隨機跳 — AI 徵信結果的一致性工程**

這是一個攸關系統可信度的關鍵工程修正：

- **發現問題**：在深度壓力測試中，我們發現同一家供應商連續查詢 3 次，雷達圖上的 5 大維度分數竟然每次不同！這在企業級決策系統中是絕對不可接受的 — 試想長官看到同一家廠商，上午評 85 分，下午變 60 分，系統的公信力就會崩塌。
- **根因追蹤 (Root Cause Analysis)**：
  - Tavily 搜尋引擎即時爬取新聞，每次回傳的文章排序與內容不同。
  - Gemini AI 生成的摘要措辭每次不同，觸發不同的情緒關鍵字。
  - 這些「隨機性」層層堆疊，導致 ESG、聲譽、交期分數全都跟著浮動。
- **解決方案：雙層快取架構 (Dual-Layer Cache)**：
  - 第一層：**記憶體快取**，同一 Session 毫秒級回傳。
  - 第二層：**SQLite 持久化快取**，伺服器重啟後 24 小時內仍穩定。過期後自動刷新最新情報。
- **成效**：三次連續查詢 Apple Inc.，五大維度分數完全一致 (`rep=85, fin=95, del=90, esg=92, pri=90`)，第二、三次查詢回應速度從 8 秒降至 **< 0.1 秒**。

## 頁面 6：讓等待變成期待 — AI 掃描動畫 (Loading UX)

**標題：使用者等 8 秒不焦慮的秘密 — 科技感 Loading 動畫**

AI 徵信掃描需要串接多個國際 API（OpenCorporates、Tavily、Gemini），平均耗時 5~10 秒。我們不把這段時間浪費：

- **挑戰**：點擊掃描按鈕後，右側面板一片空白長達 8 秒，使用者會以為系統當機。
- **解法 — 指紋雷達掃描動畫**：
  - 雙向旋轉的同心圓環 + 脈衝發光的指紋圖示 + 掃描射線，打造如同電影中 AI 分析目標的科技感。
  - 逐步進度提示：「連線全球實體註冊庫 → 掃描國際制裁黑名單 → 彙整五年負面新聞 → 生成風險雷達」，讓使用者明確知道系統正在進行多層次深度驗證。
- **心理學效應**：研究顯示，有明確進度回饋的等待，使用者感知時間會縮短 40%。我們將「漫長的等待」轉化為「令人期待的 AI 分析過程」。

## 頁面 7：支撐 AI 發展的強大後盾 (Architecture)

**標題：100% 隨插即用的 API 驅動架構**

我們知道模型訓練組正在如火如荼地優化算法，為此，UI 組打造了最友善的開發環境：
- **前後端完全分離 (Decoupled Architecture)**：
  - 前端 UI (HTML/JS) 絕對不綁死任何 AI 運算邏輯，雙方僅透過輕量級的 RESTful API 交換 JSON 資料。
- **A/B Testing 無縫接軌**：
  - 由於我們採用高度模組化的設計，未來模型組無論是要導入 `Model_A` 或 `Model_B`，甚至從機器學習換成深度學習，只要 API 欄位格式不變，直接 Push 到 GitHub `main` 分支上，前端保證**一秒無痛接軌**。
  - 我們真正做到了讓 AI 科學家能專心煉丹，UI 工程師專心刻畫體驗！
